THIS IS LESSON ABOUT DOM Существует такое понятие, как браузерное окружение, есть обьекты которые работают конкретно с браузером до этого изучали просто числый язык-JavaScript Сам по себе язык бесполезен Браузерный язык позволяет работать с сетью, так же с елементами браузера В Node.js так же будет работать классический javascript там нет браузерного окружения, можем взаимодействовать с файлами тоже с сетью браузерное окружение всегда изолировано он никогда не выходит за рамки своей вкладки Окружение- это какая-то возможность что-то сделать JavaScript- чистый язык сам по себе DOM- document object model- работа с тегами и всем что касается html и его api BOM-browser object model- позволяет узнать что это за браузер, какой это экран, где мы находимся на какой странице на какой вкладке посмотреть историю отправить запрос Существует глобальный обьект window это огромный обьект, он глобальный, т.е. доступный с самого начала, с самого верха alert()- функция обьекта виндов любая глобальная функция является частью обьекта виндов window- обьект который содержит в себе все браузерное окружение, написав функцию, могу найти ее через виндов Любое свойство window является глобальным, может быть доступно либо через обращение к обьекту, либо просто через само по себе свойство API (от английского Application Programming Interface, программный интерфейс приложений) – специальный интерфейс, который могут использовать разработчики. Главная цель использования API — внедрение в собственный продукт функций стороннего приложения. location отвечает за url часть того, где мы находимся history- просмотр истории браузера XMLHttpRequest- это возможность куда-то послать запрос и что-то получить в ответ Работа с DOM- это работа с тегами мы можем выбирать елементы из html, также их изменять что угодно сделать с html, менять классы и тд DOM-дерево Основой HTML-документа являются теги. браузер сам по себе частично может фиксить html, если в хед запихнуть диив он его отобразит первым Навигация по DOM-елементам document- ключевой глобальный обьект мы можем учитывать текстовые узлы между тегами, а также можем их игнорировать .parentElement- совйство позволяющее узнать родителя елемента, можно обращаться сколько угодно раз, отсутсвует у html .querySelector('')- метод, который присутствует у каждого html документа, можно с его помощью находить нужный елемент CSS-SELECTOR можем найти, обратиться к любому document.querySelectorAll('')- обращается сразу ко всем елементам, которые мы хотим, например ко всем лишкам единственный, кто имеет метод, один- forEach Возвращает нам - псевдомассив-коллекцию, по сути тоже список елементов, но он неимеет методов перебора и как таковых, свойств document.getElementById-возвращает только елемент по id, соответсвенно 1 елемент, существует только у document Document Object Model, сокращённо DOM – объектная модель документа, которая представляет все содержимое страницы в виде объектов, которые можно менять. Объектная модель браузера (Browser Object Model, BOM) – это дополнительные объекты, предоставляемые браузером (окружением), чтобы работать со всем, кроме документа. matches Метод elem.matches(css) ничего не ищет, а проверяет, удовлетворяет ли elem CSS-селектору, и возвращает true или false. elem.getElementsByTagName(tag) ищет элементы с данным тегом и возвращает их коллекцию. Передав "*" вместо тега, можно получить всех потомков. elem.getElementsByClassName(className) возвращает элементы, которые имеют данный CSS-класс. document.getElementsByName(name) возвращает элементы с заданным атрибутом name.
