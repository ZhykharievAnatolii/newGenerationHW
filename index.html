Функции всегда что-то возвращают Для того, чтобы функция работала, мы обязательно должны ее вызвать Сама фунция может быть значением переменной const name =getCurrency- эта функция записана в значение переменной Вид function declaration- может подыматься вверх Может быть использована до ее определения function CalcSum(){ } NaN!==NaN Потому что NaN- это результат бессмысленного вычисления Потому что один результат бессмысленный не равен другому содержит в себе название, которое глагол, что-то выполняем, в кргулых скобках содержатся аргументы() Функция ограничивает зону видиости переменных, те что в ней, существуют только в ней Аргументы-локальные переменные, содержащиеся внутри функции Функция обладает полным доступом к внешним переменным и может изменять их значение. Переменные, объявленные снаружи всех функций, такие как внешняя переменная userName в вышеприведённом коде – называются глобальными. Глобальные переменные видимы для любой функции (если только их не перекрывают одноимённые локальные переменные). Желательно сводить использование глобальных переменных к минимуму. В современном коде обычно мало или совсем нет глобальных переменных. Хотя они иногда полезны для хранения важнейших «общепроектовых» данных. Если параметр не указан, то его значением становится undefined. Имена переменных и функции не должны повторяться Ключевое слово для функции- возврат чего-то Return У return самый низкий приоритет Return используется только в функциях Return завершает работу функции, после нее уже не работает все что есть после написано в функции Return всегда работает с возвращаемым значением Значение аргумента можно сразу присваивать, когда пишем функцию Если я в функции сразу передал значение аргумента, это аргумент по умолчанию В строгом режиме, когда Function Declaration находится в блоке {...}, функция доступна везде внутри блока. Но не снаружи него. Function Expression- не может подыматься вверх, работает только там где ее записали Может быть записана непосредственно в переменную Не может подыматься вверх, потому что она записана в переменную и она не VAR, т.е. будет работать только там, где переменная arrow functions Стрелочные функции очень удобны для простых действий, особенно для однострочных. Они бывают двух типов: Без фигурных скобок: (...args) => expression – правая сторона выражения: функция вычисляет его и возвращает результат. Скобки можно не ставить, если аргумент только один: n => n * 2. С фигурными скобками: (...args) => { body } – скобки позволяют нам писать несколько инструкций внутри функции, но при этом необходимо явно вызывать return, чтобы вернуть значение
